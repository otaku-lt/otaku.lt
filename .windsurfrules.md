# Otaku.lt - Project Rules

## Tech Stack
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS 3 + shadcn/ui components
- **UI Components**: Radix UI primitives + custom components
- **State**: React hooks (useState, useMemo, useEffect, useCallback)
- **Icons**: Lucide React
- **Calendar**: FullCalendar v6 (react wrapper)
- **Data**: YAML files (js-yaml for parsing)
- **Forms**: React Hook Form + Zod validation
- **Charts**: Recharts

## Architecture
- **App Router**: `app/` directory with page.tsx files
- **Static Export**: `next.config.js` has `output: 'export'` for static site generation
- **No API routes**: Data comes from YAML files loaded at build time via `lib/events.ts`
- **Components**: `components/` for UI, `components/ui/` for shadcn components
- **Types**: `types/` directory (Event, CalendarEvent, Screening, etc.)
- **Config**: `config/` directory (event-categories.ts, etc.)
- **Data**: `data/events/` directory
  - **Monthly events** (`otaku.lt/events`): `data/events/monthly/YYYY-MM.yaml` (e.g., `2026-05.yaml`, `2026-06.yaml`)
  - **Korniha band page** (`otaku.lt/korniha`): `data/events/korniha.yaml` (separate events with setlist info)
- **Images**: `public/images/events/YYYY/MM/YYYY-MM-event-slug.jpg`
  - Events: `/images/events/YYYY/MM/YYYY-MM-event-name.jpg`
  - Korniha covers: `/images/events/YYYY/MM/YYYY-MM-event-name.jpg`
- **Scripts**: `scripts/` directory for build utilities

## Event Data Structure
Events are stored in YAML with these key fields:
- `id` (unique across ALL events globally)
- `title`, `date`, `time`, `location`, `description`
- `category` (convention, screening, concert, meetup, etc.)
- `featured` (boolean for homepage display)
- `image` (path in `/images/events/...`)
- `link` or `links[]` (booking URLs)
- `screenings[]` for movies with multiple dates/cinemas
- `setlist` for band performances (Japanese, Lithuanian, or mixed)
- `startDate` / `endDate` for multi-day events

**Critical Rule**: Event IDs must be **unique globally** across all monthly YAML files.

## Docker / Devcontainer
- Docker stack is in `.devcontainer/` directory
- **docker-compose.yml**: Defines `traefik` (reverse proxy) + `otaku-lt` (Next.js dev server)
- **Dockerfile**: Node.js 20 (TypeScript devcontainer image)
- **Ports**: 3000 (Next.js), 3001 (HMR), 8080 (Traefik dashboard)
- **Local domain**: `otaku.localhost`
- To launch: `docker compose -f .devcontainer/docker-compose.yml up -d`

## CI/CD & Deployment
- **Platform**: GitHub Actions → Cloudflare Workers
- **Workflow**: `.github/workflows/deploy.yml`
- **Command**: `wrangler deploy` (not `--env=production`, deploys to base worker `otaku-lt`)
- **Worker name**: `otaku-lt` (set in `wrangler.toml`)
- **Routes**: `otaku.lt/*` and `www.otaku.lt/*` (managed in Cloudflare dashboard)
- **Static assets**: Served from `out/` directory via `wrangler.toml` `[assets]` config
- **SPA routing**: `not_found_handling = "single-page-application"`

## Key Files
- `app/page.tsx` - Homepage with featured events banner
- `app/events/page.tsx` - Event calendar/list with filtering
- `lib/events.ts` - Event data loading and utilities
- `types/event.ts` - Event type definitions
- `config/event-categories.ts` - Category definitions with icons/colors
- `wrangler.toml` - Cloudflare Workers config

## Important Notes
- This is a **static site** (no database, no API routes)
- Event data is loaded at build time from YAML files
- Images for events are stored in `public/images/events/` and committed to git
- The `_routes.json` file handles SPA routing on Cloudflare
- `middleware.ts` handles URL rewrites for Cloudflare compatibility
